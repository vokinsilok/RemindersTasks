# Техническое задание: Telegram-бот напоминаний и задач

## 1. Цель проекта

Создать Telegram-бота для персонального управления напоминаниями и задачами.

Бот должен позволять нескольким пользователям независимо друг от друга:

- создавать собственные категории напоминаний;
- создавать разовые и повторяемые напоминания;
- получать уведомления в Telegram в нужную дату и время;
- вести список задач;
- смотреть краткий dashboard по задачам и ближайшим напоминаниям.

## 2. Стек

- Python 3.11+
- aiogram 3
- SQLAlchemy 2 async ORM
- PostgreSQL
- asyncpg
- Alembic
- Pydantic v2
- pydantic-settings
- python-dateutil
- Docker Compose

SQLite не используется как целевая БД. Для локальной разработки PostgreSQL и бот поднимаются через Docker Compose.

## 3. Архитектурные правила

- SQLAlchemy-модели описывают таблицы и связи.
- Pydantic-схемы описывают входные команды, DTO и read-модели.
- Репозитории принимают валидированные Pydantic-схемы для создания сущностей.
- Все пользовательские сущности обязаны иметь `user_id`.
- Схема БД меняется через Alembic-миграции.
- Обычный запуск бота не должен автоматически менять структуру БД.
- Docker-сервис бота перед стартом выполняет `alembic upgrade head`.

## 4. Основные сущности

### 4.1 Пользователь

Пользователь создается автоматически при `/start`.

Поля:

- `id`
- `telegram_id`
- `username`
- `first_name`
- `last_name`
- `created_at`
- `updated_at`

### 4.2 Категория напоминаний

Категория принадлежит одному пользователю.

Поля:

- `id`
- `user_id`
- `title`
- `created_at`
- `updated_at`

Ограничения:

- у одного пользователя не должно быть двух категорий с одинаковым названием;
- категории разных пользователей не пересекаются.

### 4.3 Напоминание

Напоминание принадлежит пользователю и может быть привязано к категории.

Поля:

- `id`
- `user_id`
- `category_id`
- `title`
- `description`
- `remind_at`
- `next_run_at`
- `recurrence`
- `is_active`
- `created_at`
- `updated_at`

Типы повторения:

- `once` - один раз;
- `daily` - каждый день;
- `weekly` - каждую неделю;
- `monthly` - каждый месяц;
- `yearly` - каждый год.

Логика:

- разовое напоминание после отправки становится неактивным;
- повторяемое напоминание после отправки получает новый `next_run_at`;
- scheduler восстанавливается после перезапуска бота, читая `next_run_at` из PostgreSQL.

### 4.4 Задача

Задача принадлежит одному пользователю.

Поля:

- `id`
- `user_id`
- `title`
- `description`
- `due_at`
- `priority`
- `status`
- `created_at`
- `updated_at`
- `completed_at`

Статусы:

- `todo`
- `in_progress`
- `done`

Приоритеты:

- `low`
- `normal`
- `high`

## 5. Pydantic-схемы

Минимальный набор:

- `UserRead`
- `TelegramUserUpsert`
- `CategoryCreate`
- `CategoryRead`
- `ReminderCreate`
- `ReminderUpdate`
- `ReminderRead`
- `TaskCreate`
- `TaskUpdate`
- `TaskRead`

Схемы лежат отдельно от ORM-моделей в `app/schemas/`.

## 6. Пользовательские сценарии

### 6.1 Старт

Пользователь отправляет `/start`.

Бот:

- создает или обновляет пользователя;
- показывает главное меню.

### 6.2 Главное меню

Кнопки:

- Dashboard
- Напоминания
- Категории
- Задачи

### 6.3 Категории

Пользователь может:

- посмотреть список категорий;
- создать категорию;
- удалить категорию.

Удаление категории не удаляет напоминания, а отвязывает их от категории.

### 6.4 Напоминания

Пользователь может:

- посмотреть активные напоминания;
- создать напоминание;
- выбрать категорию;
- ввести заголовок;
- ввести описание;
- выбрать тип повторения;
- ввести дату и время;
- отключить напоминание.

Формат даты на этапе MVP:

- `ДД.ММ.ГГГГ ЧЧ:ММ`, например `15.06.2026 09:30`;
- `ДД.ММ ЧЧ:ММ`, в этом случае используется текущий год.

### 6.5 Задачи

Пользователь может:

- посмотреть список активных задач;
- создать задачу;
- указать заголовок;
- указать описание;
- указать дедлайн или пропустить его;
- указать приоритет;
- перевести задачу в статус `done`.

### 6.6 Dashboard

Dashboard показывает:

- задачи на сегодня;
- просроченные задачи;
- активные задачи;
- ближайшие напоминания.

## 8. План реализации MVP

1. Подготовить зависимости и конфигурацию PostgreSQL.
2. Создать SQLAlchemy-модели.
3. Создать Pydantic-схемы.
4. Создать Alembic-миграции.
5. Добавить Dockerfile и docker-compose для PostgreSQL и Telegram-бота.
6. Создать слой репозиториев.
7. Реализовать главное меню и регистрацию пользователя.
8. Реализовать категории.
9. Реализовать создание и просмотр напоминаний.
10. Реализовать scheduler отправки напоминаний.
11. Реализовать задачи.
12. Реализовать dashboard.
13. Проверить импорт и базовый запуск приложения.

## 9. Что не входит в первый MVP

- WebApp-интерфейс;
- сложные cron-выражения;
- командная работа над общими задачами;
- вложенные подзадачи;
- вложения и файлы;
- timezone-настройки на пользователя.
